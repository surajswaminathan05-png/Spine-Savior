SPINE SAVIOR: A WEARABLE MULTI-SENSOR BIOFEEDBACK SYSTEM FOR REAL-TIME SPINAL POSTURE MONITORING, ON-DEVICE MACHINE LEARNING CLASSIFICATION, AND HAPTIC CORRECTIVE FEEDBACK


PURPOSE

The goal of this project is to design and build a wearable device that detects poor posture in real time and immediately alerts the user to correct it — without relying on cameras, cloud services, or any internet connection. Existing commercial posture devices use only a single sensor and cannot distinguish which part of the spine is misaligned. Spine Savior solves this by independently monitoring the neck (cervical), upper back (thoracic), and lower back (lumbar) regions of the spine using three separate motion sensors, then classifying the specific type of postural deviation using a personalized machine learning model trained directly on the user's own body.


PROCEDURE AND METHODS

Hardware: We built a wearable sensor harness containing three Bosch BNO055 9-degree-of-freedom inertial measurement units (IMUs). Each BNO055 combines a triaxial accelerometer, triaxial gyroscope, and triaxial magnetometer with an onboard ARM Cortex-M0+ processor that runs a sensor fusion algorithm (extended Kalman filter) to output absolute orientation as Euler angles (heading, pitch, and roll). Because all three sensors share the same default I2C address (0x28), we use a Texas Instruments TCA9548A 8-channel I2C multiplexer to route communication to one sensor at a time. An Arduino Nano 33 BLE microcontroller (built on the Nordic nRF52840 system-on-chip with an ARM Cortex-M4F processor and integrated Bluetooth Low Energy 5.0 radio) polls all three sensors at 60 Hz, packs the nine orientation values into a 36-byte binary payload, and transmits the data wirelessly via BLE to the user's computer. Two 10mm-by-3mm flat coin-type eccentric rotating mass (ERM) vibration motors operating at 3V DC and approximately 12,000 RPM are mounted at the cervical and lumbar positions on the harness. These are driven through NPN transistor switching circuits with flyback diodes for back-EMF protection. All electronics are housed in custom snap-fit enclosures designed in OpenSCAD and 3D-printed via FDM.

Software and 3D Visualization: We developed a browser-based web application using plain HTML, CSS, and JavaScript. The Web Bluetooth API connects directly to the Arduino with no drivers or middleware required. Using the Three.js graphics library, the application renders a high-fidelity, anatomically accurate 3D model of the human spine (GLB format) containing 24 individually articulating vertebrae. To translate three discrete sensor readings into smooth, biologically realistic motion across all 24 vertebrae, we implemented three layers of biomechanical computation. First, Gaussian falloff blending distributes each sensor's influence across nearby vertebrae using a normal distribution weighting function, so that movement fades naturally rather than producing sharp bends. Second, per-vertebra flexibility weighting scales each vertebra's contribution based on PubMed-published segmental range-of-motion (ROM) data, reflecting the fact that certain vertebrae (such as L4-L5 or C1-C2) are inherently more mobile than others. Third, ROM clamping enforces hard anatomical limits at every spinal level (for instance, C1-C2 allows 40 degrees of axial rotation while a lumbar level allows only about 2 degrees), preventing the model from bending in biologically impossible ways. Finally, a critically damped second-order spring-damper system (stiffness K = 150, damping D = 2 times the square root of K) smooths all rotations to eliminate sensor noise and produce fluid, natural motion.

Machine Learning: We built a posture classification system using TensorFlow.js, Google's in-browser machine learning library. A guided training wizard walks the user through holding five postures (Normal, Forward Head, Slouching, Lordosis, and Lateral Tilt) for 30 to 90 seconds each. At 60 frames per second, this produces 1,800 to 5,400 labeled data frames per posture, each consisting of a 9-dimensional feature vector (heading, pitch, and roll from each of the three sensors). These frames are segmented into overlapping sliding windows of 60 frames (1 second) with 50 percent overlap, then fed into a 1-Dimensional Convolutional Neural Network (1D-CNN) with approximately 69,500 trainable parameters. The architecture consists of two convolutional layers (32 and 64 filters), max pooling, a 64-unit dense layer with 30 percent dropout for regularization, and a 5-class softmax output. Training uses the Adam optimizer with categorical cross-entropy loss over 20 epochs. Because the model is trained on each individual user, it learns that specific person's anatomy and posture patterns rather than relying on a generic population model. All training and inference run entirely in the browser via WebGL-accelerated tensor operations. No data ever leaves the device.

AI Coaching and Haptic Feedback: When the classifier detects poor posture, two feedback mechanisms activate. First, Google Gemini Nano, a large language model embedded directly in Chrome, generates a personalized 1-to-2-sentence coaching tip based on the detected posture type and the exact real-time sensor angles, all computed locally on the user's device with no cloud processing. If Gemini Nano is unavailable, the system falls back to a curated set of 15 expert-authored coaching tips. Second, the haptic vibration motors physically vibrate at the relevant body region, alerting the user even when they are not looking at the screen. The vibration pattern uses a double-pulse (200ms on, 100ms off, 200ms on) for moderate deviations and sustained vibration for severe deviations, with a 30-second cooldown to prevent sensory habituation.


DATA AND RESULTS

The system achieves a sensor polling rate of 60 Hz, a BLE data transmission rate of 60 Hz (one 36-byte packet every 16 milliseconds), and a 3D rendering frame rate of 60 frames per second. The machine learning classifier, when trained on an individual user with at least 30 seconds of data per posture class, produces classification accuracy consistently above 90 percent as measured on a 20-percent held-out validation set. The posture scoring algorithm produces scores from 0 to 100 normalized against clinical ROM data, providing a quantitative and reproducible measure of postural alignment. The haptic feedback loop closes within approximately 1 second of a postural deviation occurring.


CONCLUSIONS

Spine Savior successfully demonstrates that a low-cost, fully self-contained, multi-sensor wearable system can perform clinically-informed spinal posture monitoring with personalized machine learning classification and multimodal corrective feedback — all without any cloud infrastructure. The key innovations are: (1) multi-regional distributed IMU sensing that independently tracks the cervical, thoracic, and lumbar spine, unlike commercial single-sensor devices; (2) biomechanically constrained 3D visualization using PubMed-sourced segmental range-of-motion data; (3) a personalized on-device 1D-CNN classifier trained on the individual user's unique body geometry; (4) a complete zero-cloud architecture ensuring biometric data privacy; and (5) a multimodal feedback loop combining visual, cognitive, and somatosensory (haptic) channels to maximize corrective response regardless of the user's attentional state.
